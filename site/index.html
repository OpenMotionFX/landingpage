The same move. Every take.

OpenMotionFX is an open-source, modular motion control system for smartphone filmmaking and VFX. Motorized camera moves you can replay exactly — the trick behind clean plates, crowd duplication and impossible transitions — for under ~$400 in parts instead of $68,000.

Why repeatability

One trick unlocks a whole visual language

Repeatable camera motion — the ability to run the exact same move twice — is the foundation of multi-pass VFX.

Shoot a pass with your actor, run the identical move on the empty set, and the two takes composite perfectly: your actor can now vanish mid-shot, play their own twin, or walk through a crowd made of six people filmed six times. Because the camera path is data, not muscle memory, every pass lines up sub-pixel.

Until now that ability started at professional robot arms. Consumer sliders can't repeat a move; prosumer gear locks repeatability behind apps and subscriptions.

How it works

Wi-Fi jitter can't ruin a take

Moves are compiled on your laptop and pre-loaded into every module. A single broadcast GO signal — synchronized to within a tenth of a millisecond — starts them all. The network never carries real-time motion, so each module executes its move on precise local timing, every single time.

Compile

The Conductor app on your laptop turns your keyframes into exact per-motor trajectories — all the heavy math happens up front, not mid-move.

Upload

Each module receives only its own portion of the move and confirms it loaded correctly. No silent failures.

Arm

Modules home themselves, travel to their start marks, and report ready — a checklist, just like a real set.

Go

One broadcast beacon starts every module against a shared clock. Motors, lights and camera triggers execute in lockstep, and encoders record what actually happened.

The system

Eight modules. One wireless language. None mandatory.

Every module runs the same brain (a ~$6 ESP32-S3) and speaks the same open protocol, so they stack into one synchronized rig. A lone pan/tilt head is a complete product — everything else is additive.

Pan/Tilt Head The flagship. Brushless motors, zero-backlash printed gears, repeatable to 0.02°. $75–95 Slider 500 mm linear rail, belt-driven, repeatable to 0.05 mm. $90–110 Dolly Floor track with positive drive — repeats within 0.5 mm over 3 m of track. $80–110 Turntable Product and prop spins, identical every time. The cheapest way in. $40–55 Focus/Zoom Motor Makes focus an axis like any other — repeatable focus pulls, recorded in the move file. $35–50 Key Light High-CRI COB light, flicker-free at any shutter angle, cues synced to the move. $55–75 FX Light Addressable pixels for repeatable lightning, fire, TV flicker and police strobes. $30–45 Sync Box Camera triggers, sync flash, tally and timecode — the rig's timekeeper. $25–35

Every take is data

Shoot it, then open it in Blender

Every take produces a move file: the camera's position, rotation, lens settings and lighting state for every frame. Import it into Blender or Unreal and your virtual camera matches the real one — no motion tracking, no guesswork.

It works both ways: design a move in Blender, and the hardware performs it on set. Previz becomes the shot.

Design principles

Six rules the whole project obeys

Repeatability over speed

A phone weighs less than 300 g. We spend the torque surplus on precision, not whip-pans.

The network is untrusted

All timing-critical execution is local to each module. Wi-Fi carries plans, never motion.

Boring parts, clever geometry

NEMA17 motors, GT2 belts, standard rails — the 3D-printer supply chain. The novelty lives in printed parts and firmware.

Every take is data

If it happened on set, it lands in the move file — camera, lens, lights, all of it.

No module is mandatory

A lone pan/tilt head is a complete product. Everything else is additive.

Documentation is a deliverable

Bad docs are a top complaint against commercial gear. Ours are part of the spec.

Honest status

Designed in the open, built in the open

OpenMotionFX is currently in the planning and bench-prototyping phase. The full system architecture, hardware specs, software design and VFX pipeline are written and public — the first physical prototypes come next. No hardware is for sale, and nothing on this site claims otherwise.

The plan is phased so that every phase ends with something a filmmaker can actually shoot with, starting with the pan/tilt head.

FAQ

DIY motion control, phone VFX and camera robots

Straight answers for the searches that usually come before a build: cheap repeatable camera moves, clean plates, Blender camera data, and open-source alternatives to closed motion control gear.

What is the cheapest way to get repeatable camera moves for VFX?

OpenMotionFX is designed as a DIY, open-source motion control system that targets roughly $400 in parts for a full modular rig. It focuses on repeatable camera moves for clean plates, crowd duplication, twinning, product shots and other multi-pass VFX work.

How do I shoot clean plates, crowd duplication or twin shots with a phone?

You need a camera move that can repeat across multiple takes. OpenMotionFX is designed to run the same pan, tilt, slider, dolly, focus and lighting cues again and again, so a phone shot with an actor can line up with an empty clean plate or another performance pass.

What is OpenMotionFX?

OpenMotionFX is an open-source, modular motion control system for smartphone filmmaking and VFX. It uses ESP32-S3 wireless modules, a laptop Conductor app and planned .omfx move files to connect physical camera motion with post-production tools.

Is OpenMotionFX a DIY alternative to expensive motion control rigs?

Yes, that is the goal. OpenMotionFX is built around accessible maker parts instead of closed commercial motion control hardware, while staying honest that the project is still in planning and bench prototyping rather than finished production hardware.

How does OpenMotionFX avoid Wi-Fi jitter during a camera move?

Wi-Fi carries the plan, not the live motion. The laptop Conductor app compiles the move first, uploads each module's instructions, waits for every node to arm, then sends a small GO beacon. Each module executes locally from its own timing, so a network hiccup does not steer the motors mid-shot.

Can OpenMotionFX export camera tracking data to Blender or Unreal Engine?

That is a core goal. Every take is planned to produce an .omfx move file containing camera position, rotation, lens metadata, lighting cues and performed encoder data. The intended pipeline includes Blender, Unreal Engine, FBX, USD, JSON and CSV-friendly exports.

Which OpenMotionFX module should I build first?

The pan and tilt head is the intended first build because it can be useful by itself for repeatable nodal pans, phone VFX plates, product moves and controlled transitions. Other modules such as the slider, dolly, turntable, focus motor, lights and sync box can be added later.

Is OpenMotionFX available to buy now?

No. OpenMotionFX is in the planning and bench-prototyping phase. The public work right now is architecture, module planning, protocol design, software design and early hardware validation. No finished hardware is for sale.

Is OpenMotionFX cloud-based or subscription-based?

No. The project is designed around a local-first workflow: no cloud account, no subscription gate and no locked app ecosystem. The license intent is open software, open hardware, open documentation and a public protocol.
